Buatkan Issue.md yang berisi perencanaan untuk nanti di implementasikan oleh junior programmer atau AI model yang lebih murah

Isi dari Perencanaannyan adalah sebagai berikuit
buat table users:
id integer auto increment
name varchar 255 not null
email varchar 255 not null
password varchar 255 not null ( password mereupakan hash dari bycript)
created_at timestampe default current_timestamp

buatkan API untuk registrasi User baru

Endpoint : POST /api/register
Body : {
    "name": "John Doe",
    "email": "[EMAIL_ADDRESS]",
    "password": "password"
}

Response : {
    "message": "User created successfully",
    "data": {
        "id": 1,
        "name": "John Doe",
        "email": "[EMAIL_ADDRESS]",
        "created_at": "2022-01-01T00:00:00.000Z"
    }
}


buatkan API untuk login User

Endpoint : POST /api/login
Body : {
    "email": "[EMAIL_ADDRESS]",
    "password": "password"
}

Response : {
    "message": "User logged in successfully",
    "data": {
        "id": 1,
        "name": "John Doe",
        "email": "[EMAIL_ADDRESS]",
        "created_at": "2022-01-01T00:00:00.000Z"
    }
}

buatkan API untuk mendapatkan semua User

Endpoint : GET /api/users
Response : {
    "message": "Users fetched successfully",
    "data": [
        {
            "id": 1,
            "name": "John Doe",
            "email": "[EMAIL_ADDRESS]",
            "created_at": "2022-01-01T00:00:00.000Z"
        },
        {
            "id": 2,
            "name": "Jane Doe",
            "email": "[EMAIL_ADDRESS]",
            "created_at": "2022-01-01T00:00:00.000Z"
        }
    ]
}


Struktur Folder di dalam src
- routes: ini berisi routing elysia js
- services : ini berisi logic bisnis aplikasi

Struktur file 
- routes: menggunakan format misal users-route.ts
- services : menggunakan format misal users-service.ts
